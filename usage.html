API Reference
NyxoAI Assistant API
The NyxoAI Assistant API gives you programmatic access to every model in the NyxoAI catalogue. Send a message, pick a model, and get a response — with optional memory injection, skill prompts, live web crawling, side-by-side model comparison, and granular thinking control.
Persistent Memory
Inject conversation history so the model remembers prior context across requests.
Skill Injection
Supply domain knowledge or custom instructions to shape how the model behaves.
Web Crawling
Pass URLs and the API fetches their content at runtime, grounding answers in live data.
Model Comparison
Run the same prompt through two models simultaneously and receive both responses.
Base URL for all API requests: https://nyxoai.vortexa.cloud
Authentication
API Keys
Every request to /api/v1/assistant is authenticated via your username and an apiKey in the request body. Keys always start with nyxo_ and are tied to your account. Generate and manage them from your settings page.
Never expose your API key in public repositories or client-side code. Treat it like a password.

Creating an API key

HTTP 
POST https://nyxoai.vortexa.cloud/end/create-api-key
Content-Type: application/json

{
  "username": "your_username",
  "password": "your_password",
  "name":     "my-app-key"
}

The response is { "success": true, "name": "my-app-key", "apiKey": "nyxo_..." }. Save this key immediately — it is only shown once.

Token System
How tokens work
NyxoAI uses a token-based billing system. New accounts start with 500 tokens. Each request deducts tokens based on the byte-length of your inputs plus the model's output.
What is countedCapNotes
message20,000 charsThe main user prompt.
memory20,000 charsInjected conversation history.
skills10,000 charsCustom instruction block.
Crawled URL content50,000 chars per URLFetched at runtime.
Model outputUp to remaining balanceActual response text deducted after reply.
Check your remaining token balance at any time via GET /end/me with a valid JWT, or from your account dashboard.
Endpoint
POST /api/v1/assistant
The core inference endpoint. Send a prompt, choose a model, and optionally layer in memory, skills, crawled URLs, a compare model, thinking intensity, and temperature.
POST https://nyxoai.vortexa.cloud/api/v1/assistant
All requests must use JSON. Always set the Content-Type: application/json header.
Parameters
Request body fields
All fields are sent as a flat JSON object. Required fields must always be present.
FieldTypeRequiredDescription
username string required Your NyxoAI account username.
apiKey string required A valid API key for this account. Must start with nyxo_.
message string required The user's prompt or question. Max 20,000 characters.
modelKey string required The key of the model to use. Retrieve valid keys from GET /end/get-model-data or the Models page.
memory string optional Conversation history or persistent context injected before the prompt. Max 20,000 chars. Default: ""
skills string optional Custom instructions, persona, or domain knowledge. Injected as a skill block. Max 10,000 chars. Default: ""
think string optional Thinking intensity. One of "LOW", "MEDIUM", or "HIGH". Default: "LOW"
crawling string[] optional Array of URLs to fetch at runtime. Content is appended to the prompt context. Each URL capped at 50,000 chars. Default: []
compare string | null optional A second model key to run the same prompt through simultaneously. Both responses are returned. Default: null
temperature number optional Sampling temperature. Range 0.0 (deterministic) to 1.0 (creative). Default: 0.5
Response
Response format
A successful response is a JSON object whose keys are the model key(s) you requested and whose values are the model's text output.

Single model

JSON 
{
  "meta-llama/llama-3-8b": "The sky appears blue because of Rayleigh scattering..."
}

With compare model

JSON 
{
  "meta-llama/llama-3-8b":  "The sky appears blue because of Rayleigh scattering...",
  "mistralai/mistral-7b":   "Sunlight contains all wavelengths of visible light..."
}
Customization
Memory
The memory field lets you inject previous conversation turns or any persistent facts so the model can continue a stateful conversation. Because the API is stateless, your application stores and passes this context on every request.
Memory counts toward your token usage. Keep it trimmed by storing only the most recent exchanges or a condensed summary of the conversation.
JSON — Continuing a conversation 
{
  "username": "alice",
  "apiKey":   "nyxo_abc123...",
  "modelKey": "meta-llama/llama-3-8b",
  "message":  "What should I make for dinner?",
  "memory":   "User: I am vegetarian and allergic to nuts.\nAssistant: Got it, I will keep that in mind."
}

Format memory as a plain-text log with a consistent separator like User: / Assistant: so the model can distinguish turns. You can also store facts, preferences, or any background context here.

Customization
Skills
The skills field injects a block of custom instructions before the user's message. Use it to define a persona, restrict behavior, supply domain knowledge, or enforce a specific output format.
JSON — Customer support persona 
{
  "skills": "You are a helpful customer support agent for Acme Store. Always be polite and concise. If you do not know the answer, say so and offer to escalate. Never discuss competitors."
}
JSON — Enforce JSON output schema 
{
  "skills": "Always respond with a valid JSON object. Schema: { \"summary\": string, \"sentiment\": \"positive\" | \"neutral\" | \"negative\", \"keywords\": string[] }. No prose, no markdown fences — only raw JSON."
}
Customization
Thinking Mode
The think parameter controls how deeply the model reasons before responding. Higher thinking modes produce more thorough, step-by-step reasoning but consume more tokens.
LOW Fast, direct answers. Best for simple Q&A and chat.
MEDIUM Balanced depth. Good for summaries and analysis tasks.
HIGH Deep chain-of-thought. Best for complex reasoning and planning.
JSON 
{
  "think":   "HIGH",
  "message": "Explain the trade-offs between microservices and a monolith architecture."
}
Customization
Web Crawling
Pass an array of URLs in the crawling field. The server fetches each URL at request time (8-second timeout per URL) and appends the page content to the prompt, grounding the model's answers in live data you specify.
Each crawled page counts toward your token usage. URLs that fail (timeouts, 404s) are silently skipped. Only public, unauthenticated pages can be fetched.
JSON — Summarize a live article 
{
  "username": "alice",
  "apiKey":   "nyxo_abc123...",
  "modelKey": "meta-llama/llama-3-8b",
  "message":  "Summarize the key points of this article.",
  "crawling": ["https://example.com/some-article"]
}
Customization
Compare Models
Set compare to a second model key to run your prompt through two models at the same time. The response object contains an entry for each model key, making it easy to diff outputs side-by-side in your UI.
JSON 
{
  "username": "alice",
  "apiKey":   "nyxo_abc123...",
  "modelKey": "meta-llama/llama-3-8b",
  "compare":  "mistralai/mistral-7b",
  "message":  "Write a haiku about the ocean."
}

Input tokens are counted once, not twice. The compare model's output is tracked separately for usage stats but does not double the input deduction from your balance.

Customization
Temperature
temperature controls the randomness of the model's output. Lower values produce consistent, factual responses; higher values produce more creative and varied output.
0.0 – 0.3 Precise and consistent. Best for data extraction and factual Q&A.
0.4 – 0.7 Balanced default. Works well for general-purpose tasks.
0.8 – 1.0 Creative and varied. Great for brainstorming and writing.
JSON — Creative mode 
{
  "temperature": 0.9,
  "message":     "Give me 5 unique product names for a coffee brand."
}
Code Examples
Full examples
Ready-to-run snippets for the most common use cases.

Minimal request — JavaScript (fetch)

JavaScript 
const response = await fetch("https://nyxoai.vortexa.cloud/api/v1/assistant", {
  method:  "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    username: "alice",
    apiKey:   "nyxo_abc123...",
    modelKey: "meta-llama/llama-3-8b",
    message:  "What is the capital of France?"
  })
});

const data = await response.json();
console.log(data["meta-llama/llama-3-8b"]); // "The capital of France is Paris."

Full-featured request — JavaScript

JavaScript 
const response = await fetch("https://nyxoai.vortexa.cloud/api/v1/assistant", {
  method:  "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    username:    "alice",
    apiKey:      "nyxo_abc123...",
    modelKey:    "meta-llama/llama-3-8b",
    compare:     "mistralai/mistral-7b",     // run both models
    message:     "Explain neural networks simply.",
    memory:      "User: I am a high school student.\nAssistant: Got it!",
    skills:      "Use simple language. Avoid jargon. Keep answers under 100 words.",
    think:       "MEDIUM",
    crawling:    ["https://en.wikipedia.org/wiki/Neural_network"],
    temperature: 0.6
  })
});

const data = await response.json();
console.log("Llama:",   data["meta-llama/llama-3-8b"]);
console.log("Mistral:", data["mistralai/mistral-7b"]);

Minimal request — Python

Python 
import requests

payload = {
    "username":    "alice",
    "apiKey":      "nyxo_abc123...",
    "modelKey":    "meta-llama/llama-3-8b",
    "message":     "What is quantum entanglement?",
    "think":       "HIGH",
    "temperature": 0.4
}

resp = requests.post(
    "https://nyxoai.vortexa.cloud/api/v1/assistant",
    json=payload
)

data = resp.json()
print(data["meta-llama/llama-3-8b"])

Stateful chat loop — JavaScript

JavaScript 
let memory = "";

async function chat(userMessage) {
  const res = await fetch("https://nyxoai.vortexa.cloud/api/v1/assistant", {
    method:  "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      username: "alice",
      apiKey:   "nyxo_abc123...",
      modelKey: "meta-llama/llama-3-8b",
      message:  userMessage,
      memory:   memory       // pass accumulated history each turn
    })
  });

  const data  = await res.json();
  const reply = data["meta-llama/llama-3-8b"];

  // Append this turn so the next call has full context
  memory += `User: ${userMessage}\nAssistant: ${reply}\n`;
  return reply;
}

await chat("My name is Bob.");
await chat("What is my name?");  // => "Your name is Bob."

Error codes
All error responses follow the shape { "error": "error_code" }.
HTTP StatusError CodeMeaning
200 — Success. Response body contains model output keyed by model name.
400 nyxoAI_missing_fields One or more required fields (username, apiKey, message, modelKey) are missing.
400 nyxoAI_not_enough_tokens Your balance is too low to cover the estimated input cost. Top up your tokens to continue.
400 nyxoAI_invalid_model The modelKey does not exist in the model catalogue.
400 nyxoAI_invalid_compare_model The compare model key does not exist.
401 nyxoAI_invalid_user No account found matching the provided username.
403 nyxoAI_invalid_api_key The apiKey is not associated with this account or has been revoked.
500 nyxoAI_server_error Unexpected server-side error. Retry with exponential back-off.
